Network Analyst Developer APIs

The ArcGIS platform has multiple APIs to access Network Analyst functionality. The various APIs support a variety of workflows depending upon the area of the platform being used. A general overview of Network Analyst developer functionality can be found at in the "Find routes and directions" section of the developer documentation.

JavaScript APIs

Web applications can be built using any of several APIs. The ArcGIS API for JavaScript provides a number of classes to assist in accessing network analysis services. When using the Esri Leaflet, Mapbox GL, or the OpenLayers JavaScript libraries, network analysis services can be accessed using the ArcGIS REST JS modules. (See also the REST section below.)

REST - ArcGIS REST JS and ArcGIS API for Python

The ArcGIS network analysis services are accessible through a REST API and can be used to perform analysis against ArcGIS Online or ArcGIS Enterprise services. You can easily interact with these REST endpoints via modules provided for both JavaScript (ArcGIS REST JS) or Python (ArcGIS API for Python). They support both map services with network analysis capability and network analysis geoprocessing services.

ArcGIS Runtime APIs

The ArcGIS Runtime APIs can be used to build mobile or desktop applications. The iOS and Android APIs support building applications for their respective mobile platforms, the Java API can be use to build desktop applications for Windows or Linux, and the .Net and Qt APIs support building applications across both mobile and desktop platforms. ArcGIS Runtime applications can use either local geodatabases containing transportation networks or map services with network analysis capability (NAServer services). The ArcGIS Runtime API also includes classes to help in creating a navigation application. These classes build on top of the network analysis functionality.

ArcPy - ArcGIS Pro

Network Analyst functionality in ArcGIS Pro can be accessed through Python using the arcpy.na and the arcpy.nax modules. The arcpy.nax module is newer, whereas the arcpy.na module is our legacy implementation. The arcpy.nax is faster with a simpler API; however there are workflows where using the arcpy.na module is needed.

ArcGIS Enterprise SDK

This SDK allows developers to extend the functionality of ArcGIS Enterprise through server object extensions (SOEs) and server object interceptors (SOIs). With the ArcGIS Enterprise 10.9.1 release, the ArcGIS Enterprise SDK has expanded its interface to include access to network datasets, thus allowing SOEs and SOIs to interact with these objects.
